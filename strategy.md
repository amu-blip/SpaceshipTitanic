# Spaceship Titanic 竞赛策略规划

## 一、问题定位

- **任务类型**：二分类（预测乘客是否被传送到异次元）
- **评估指标**：分类准确率（Accuracy）
- **数据规模**：训练集约 8700 条，测试集约 4300 条
- **当前 SOTA 水平**：公开排行榜 Top 1% 约 81.5%+ accuracy，合理目标是进入前 5%（约 80.8%+）

---

## 二、数据理解与 EDA 要点

在动手建模前，优先搞清楚以下关键规律：

1. **CryoSleep 是强信号**：处于冷冻睡眠的乘客被传送的概率远高于普通乘客，且冷冻乘客的所有消费字段（RoomService、FoodCourt 等）理论上应为 0（被关在舱内）。这个规则可用于缺失值推断。

2. **消费字段是核心特征**：RoomService、FoodCourt、ShoppingMall、Spa、VRDeck 五项消费与目标强相关。消费总额为 0 的乘客被传送概率显著更高。

3. **乘客分组结构**（PassengerId = gggg_pp）：同一 group 的乘客往往是家庭成员，传送结果可能存在组内相关性。同组人数（group size）是一个隐含特征。

4. **Cabin 含有位置信息**（格式 deck/num/side）：不同甲板（A-G）、左舷/右舷（P/S）的传送率存在显著差异，需要解析拆分。

5. **Name 的姓氏可辅助确认家庭关系**：同姓氏 + 同 group 可用于缺失值的组内填充。

---

## 三、特征工程（最关键环节）

### 3.1 从 PassengerId 提取

- `GroupId`：gggg 部分，标识所属群体
- `GroupSize`：同 GroupId 的人数（1 = 独行旅客，>1 = 群体乘客）
- `IsAlone`：是否独行（布尔值）

### 3.2 从 Cabin 提取

- `Deck`：甲板层（A/B/C/D/E/F/G/T）
- `CabinNum`：舱室编号（数值，可分桶）
- `Side`：左舷（P）/ 右舷（S）

### 3.3 消费特征

- `TotalSpend`：五项消费之和
- `HasSpend`：总消费是否 > 0（布尔值）
- 各消费项对数变换（log1p），缓解右偏分布
- `SpendPerItem`：各项消费占比（消费结构特征）

### 3.4 年龄特征

- `AgeGroup`：分桶为儿童（0-12）、青少年（13-17）、成年人（18-60）、老年人（60+）
- 儿童和老年人的传送概率与其他群体不同

### 3.5 交互特征（高收益）

- `CryoSleep × TotalSpend`：冷冻 + 有消费 = 数据噪声或强异常标记
- `HomePlanet × Destination`：出发地与目的地组合
- `Deck × Side`：船舱位置组合
- `VIP × TotalSpend`：VIP 但低消费是反常信号

### 3.6 组内统计特征（泄漏风险低，信息量高）

- 同 GroupId 中其他成员的传送率均值（使用 leave-one-out 方式防止泄漏）
- 同 GroupId 的 HomePlanet、CryoSleep 一致性

---

## 四、缺失值处理策略

缺失值处理的质量直接影响最终分数，建议优先级从高到低：

1. **规则推断**（最优先）：
   - 若某乘客五项消费均为 0 且 CryoSleep 缺失 → 填充 CryoSleep=True
   - 若某乘客 CryoSleep=True 且消费字段缺失 → 填充 0

2. **组内填充**：
   - HomePlanet、Cabin 等字段，同 group 内乘客往往相同 → 用组内众数填充

3. **模型填充**（对重要特征）：
   - 用其他特征训练一个小模型预测缺失的 CryoSleep、HomePlanet 等

4. **统计填充**（兜底）：
   - 分类特征用众数，数值特征用中位数，按 HomePlanet/Destination 分层填充更精准

---

## 五、建模策略

### 5.1 基础模型（必选）

| 模型 | 优势 |
|------|------|
| **LightGBM** | 速度快、对类别特征友好、调参空间大 |
| **XGBoost** | 与 LGB 互补，集成效果好 |
| **CatBoost** | 原生处理类别特征，对小数据集鲁棒 |

以上三者是树模型的标准三件套，必须都跑一遍并做集成。

### 5.2 进阶模型（可选）

- **TabNet**：专为表格数据设计的注意力神经网络，与树模型差异化
- **FT-Transformer / SAINT**：Transformer 架构的表格模型，Top 方案常用
- **MLP with embeddings**：对类别特征做 embedding，与树模型集成效果不错

### 5.3 集成策略

- **Stacking**：一级模型的 OOF（out-of-fold）预测作为二级模型输入，二级用 LR 或小 LGB
- **Blending**：简单加权平均，权重可通过验证集上的表现调优
- 树模型 + 神经网络 的混合集成通常比纯树模型提升 0.1-0.3%

---

## 六、验证策略

- 使用 **5-Fold 或 10-Fold 分层交叉验证**（StratifiedKFold），目标标签分层
- 确保本地 CV 与公开榜分数一致（若差距大则考虑数据泄漏或分布偏移问题）
- 对组内乘客（同 GroupId），考虑 **GroupKFold** 避免组内信息泄漏到验证集

---

## 七、后处理与规则修正

模型预测完成后，可用业务规则做后处理提升准确率：

1. **组内一致性修正**：若同一 group 所有成员预测结果高度一致（如 5 人均 ≥ 0.7），可将组内少数派概率较低的预测统一修正

2. **CryoSleep 硬规则**：若 CryoSleep=True 且模型预测 False，考虑强制修正（CryoSleep 乘客被传送率约 80%+）

3. **阈值优化**：默认 0.5 并非最优，在验证集上搜索最优阈值（对 Accuracy 影响相对有限，但值得尝试）

---

## 八、进阶技巧（冲刺 Top 1%）

1. **伪标签（Pseudo-labeling）**：用高置信度的测试集预测加入训练集重新训练

2. **对抗验证（Adversarial Validation）**：检测训练集和测试集分布是否一致，若不一致则降低差异最大特征的权重

3. **Optuna / Hyperopt 自动调参**：对 LightGBM 等模型做大规模超参搜索

4. **特征选择**：用 SHAP 值或特征重要性删除噪声特征，防止过拟合

5. **种子平均（Seed Averaging）**：同一模型换不同随机种子训练多次，结果平均可减少方差

---

## 九、执行路线图

```
阶段 1（EDA + 数据清洗）
  └─ 分析各特征分布、缺失率、与目标的相关性
  └─ 可视化 CryoSleep、HomePlanet、消费字段与 Transported 的关系

阶段 2（特征工程）
  └─ 解析 PassengerId、Cabin
  └─ 构造消费聚合特征、交互特征、组内特征
  └─ 缺失值规则推断 + 统计填充

阶段 3（基线模型）
  └─ 单个 LightGBM + 5-Fold CV，建立基线
  └─ 目标：本地 CV ≥ 0.800

阶段 4（优化迭代）
  └─ 特征重要性分析 → 删除噪声、增加新特征
  └─ 加入 XGBoost、CatBoost
  └─ 目标：本地 CV ≥ 0.806

阶段 5（集成 + 后处理）
  └─ Stacking / Blending
  └─ 神经网络补充多样性
  └─ 规则后处理
  └─ 目标：公开榜 ≥ 0.808（Top 5%）

阶段 6（冲刺）
  └─ 伪标签、种子平均、阈值搜索
  └─ 目标：公开榜 ≥ 0.812（Top 1%）
```

---

## 十、注意事项

- **不要过拟合公开榜**：公开榜只包含 50% 测试数据，过度依赖公开榜调参会在私榜翻车
- **本地 CV 是最可信的指标**，以 CV 分数为主要优化目标
- **特征工程 > 模型选择**：在这个数据集上，精心设计的特征比换模型更有效
- **记录每次实验**：建议用表格记录每次改动、CV 分数、榜单分数，方便回溯最优方案
